请基于本月的周报和日报数据，生成月度趋势分析报告（草稿）。

## 输入

### 周报文件
读取 `${REPORT_DIR}/weekly/` 下本月（${MONTH_START} 至 ${MONTH_END}）的所有周报 .md 文件。
重点提取：
- 量化指标（tag 热度排名、topic share、周环比）
- 主题聚类及关键词演变
- 信号状态（trending/emerging/fading/declining）
- 关联图谱
- 注意力流向分析

### 结构化条目数据
读取 `${REPORT_DIR}/daily/` 下本月所有 .items.yaml 文件。
用于按趋势维度聚合统计：
- 每个 tag 的 source_type 分布（学术/行业/融资/专利/产品/市场）
- 每个 tag 的 source_quality 分布
- watch_entity 提及频次

### 上月月报（如存在）
读取 `${REPORT_DIR}/monthly/${LAST_MONTH}.md`，用于纵向对比。

### 配置文件
读取 `${SCAN_CONFIG}` 获取当前 domains、signals、sources 配置。

## 工作流程

### 阶段 1：多维动量评分

对本月每个主要趋势（从周报的 trending + emerging 信号中提取），计算五维动量评分（参考 McKinsey Technology Trends Outlook 方法论）：

| 维度 | 数据来源 | 计算方式 |
|------|---------|---------|
| **学术活跃度** | items.yaml 中 source_type=academic 的条目数 | 本月论文数 / 总条目数 |
| **媒体关注度** | items.yaml 中 source_type=industry 的条目数 | 本月行业报道数 |
| **投资热度** | items.yaml 中 source_type=funding 的条目数 | 融资事件数及金额（如有） |
| **产品化进展** | items.yaml 中 source_type=product 的条目数 | 产品发布/商用案例数 |
| **专利动向** | items.yaml 中 source_type=patent 的条目数 | 专利相关条目数 |

每个维度归一化为 0-5 分（相对于本月所有趋势），得出综合动量评分。

与上月对比（如有上月月报），标注动量变化方向。

### 阶段 2：Hype Cycle 阶段判定

基于动量评分 + 定性信号，对每个主要趋势判定其 Hype Cycle 阶段：

| 阶段 | 判定依据 |
|------|---------|
| **① 触发期** | 学术活跃度高，媒体刚开始关注，无商用产品，无或极少融资 |
| **② 膨胀期** | 媒体关注度激增，融资热潮，大量初创涌入，产品多为概念/原型，承诺多于交付 |
| **③ 幻灭期** | 媒体热度下降（fading 信号），初创失败案例增多，投资降温，产品未达预期 |
| **④ 爬坡期** | 二三代产品出现，头部玩家确立，实际应用案例增多，商业模式清晰 |
| **⑤ 成熟期** | 大规模采用，标准化，竞争格局稳定，渗透率高 |

同时给出 TRL 参考评估（1-9），基于该趋势中最先进的实现：
- TRL 1-3：基础研究/实验室验证
- TRL 4-6：原型/系统验证
- TRL 7-9：产品化/商用部署

**重要**：每个判定必须附带具体依据（引用哪些周报数据、哪些条目支撑了这个判断）。不确定的判定标注"置信度：低"。


### 阶段 2.5：补充搜索（Delta Retrieve）

对以下情况，进行针对性补充搜索，弥补日报/周报数据的不足：
- **Hype Cycle 判定依据不足**：搜索该技术的商业化现状、头部玩家、市场规模、失败案例等
- **动量评分中某维度数据缺失**：如投资热度维度条目为 0，搜索该领域近期融资信息
- **新兴趋势的背景**：对本月首次出现的 emerging 信号，搜索其技术原理、发展历史、关键玩家
- **竞争格局**：对 trending 信号，搜索当前市场竞争格局和标准化进展

补充搜索的结果在报告中标注为"月报补充搜索"来源，与周报/日报原始数据区分。

**论文阅读限制**：补充搜索中如涉及学术论文，浏览摘要不限篇数，但精读全文最多 `budget.paper.monthly_deep_read` 篇，优先选择文献综述（review/survey）。

### 阶段 3：纵向趋势分析

回答核心问题：**本月什么在结构性地改变？**（参考 LatentWire 月报视角）

- **趋势加速**：哪些趋势的动量评分在持续上升？加速的驱动力是什么？
- **趋势转折**：哪些趋势出现了阶段迁移（如从触发期进入膨胀期）？转折的标志事件是什么？
- **新兴方向**：本月有哪些全新的技术方向首次出现？它们的潜在影响有多大？
- **消退方向**：哪些上月的热点本月显著降温？是暂时回调还是趋势终结？
- **跨领域融合**：不同领域的趋势之间是否出现了新的交叉点？

### 阶段 4：交叉影响与场景推演

参考 environmental-scanning-foresight 的方法论：

**4a. 交叉影响矩阵**
选取本月 top 5 趋势，分析两两之间的交互关系（强化/对冲/因果），形成交叉影响矩阵。

**4b. 关键不确定性识别**
从所有趋势中识别：
- **确定性要素**（高影响 + 低不确定性）：可以直接纳入规划
- **关键不确定性**（高影响 + 高不确定性）：需要场景规划

**4c. 未来 3-6 个月展望**
基于当前趋势和交叉影响，给出 2-3 个可能的发展情景（不是预测，是"如果...那么..."的推演）。

### 阶段 5：生成 scan-config 修改建议

基于以上分析，生成配置修改建议：

- **domains 权重调整**：哪些领域应该升权/降权，依据是什么
- **keywords 更新**：哪些关键词应该新增/删除/细化
- **watch_entities 更新**：哪些公司/机构应该新增/移除关注
- **sources 更新**：哪些信息源应该新增/降级/移除
- **新增领域建议**：是否有新的领域值得单独跟踪

## 约束

- 月报是草稿，所有判定必须附带依据和置信度
- 不确定的判断宁可标注"置信度：低"也不要给出过于肯定的结论
- Hype Cycle 阶段判定必须基于本月实际数据，不要基于常识或先验知识
- 如果本月周报不足 3 篇，在执行摘要中说明数据量有限
- 不要编造数据或虚构趋势
- config 建议文件中每项修改必须附带理由
