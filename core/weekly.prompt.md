请基于本周的日报，生成周度信号地图。

## 输入

### 日报文件
读取 `${REPORT_DIR}/daily/` 下本周（${WEEK_START} 至 ${WEEK_END}）的所有 .md 日报文件。
解析每篇日报的：
- YAML frontmatter（tags、weak_signals、source_count、item_count）
- 正文中的各条目及其信号标签
- "趋势信号"章节
- "信息源评估"章节

### 日报结构化数据
读取 `${REPORT_DIR}/daily/` 下本周所有 .items.yaml 文件。
用于精确统计每个 tag 的 source_type 分布和 watch_entity 提及。

### 配置文件
读取 `${SCAN_CONFIG}` 获取：
- `domains`：当前搜集领域及权重
- `signals`：上周的信号追踪状态（trending/emerging/fading/declining）
- `sources`：固定信息源列表

### 上周周报（如存在）
读取 `${REPORT_DIR}/weekly/${LAST_WEEK}.md`（如存在），用于计算周环比变化。

## 工作流程

### 阶段 1：数据提取与量化

从本周所有日报中提取结构化数据：

**1a. Tag 频率统计**
统计每个 tag 在本周日报中出现的天数和总次数，计算：
- 本周出现天数 / 总天数 = 覆盖率
- 本周 topic share = 该 tag 出现次数 / 所有 tag 总出现次数（百分比，反映注意力占比）
- 与上周周报对比（如有）：周环比变化（↑ 上升 / → 持平 / ↓ 下降 / 🆕 新增 / ❌ 消失），标注具体变化幅度（如 +3.2%）

**1b. Watch Entities 提及追踪**
统计 scan-config 中 watch_entities 各实体本周被提及的次数和上下文。

**1c. 弱信号汇总**
汇总本周所有日报 frontmatter 中的 weak_signals 字段。

**1d. 信息源效能统计**
汇总本周各日报的"信息源评估"章节，统计：
- 哪些固定信息源持续提供有价值内容
- 哪些信息源被多次建议新增
- 哪些信息源多次未能提供有效内容


### 阶段 2：事件聚类与去重

将本周所有日报条目按**主题/事件**聚类，而非按天罗列：
- 同一事件在多天出现 → 合并为一个条目，标注持续天数和演变过程
- 相关但不同的事件 → 归入同一主题簇
- 每个主题簇给出一句话概括
- 对跨周持续的主题簇，标注**主题内关键词演变**：本周 vs 上周同一主题的代表性关键词有什么变化（如：上周侧重"原理验证"，本周转向"商业化"）

### 阶段 2.5：关联分析

在聚类基础上，识别主题簇之间的关联关系（参考 environmental-scanning-foresight 的 cross-impact analysis）：

- **强化关联**：趋势 A 和趋势 B 同时出现且相互加速（如：某材料突破 + 某公司融资 → 指向同一技术方向）
- **对冲关联**：趋势 A 和趋势 B 存在张力（如：国产替代加速 vs 国际巨头降价）
- **因果链条**：趋势 A 可能触发趋势 B（如：新传感器原理论文发表 → 半年后出现相关初创公司）

对识别出的关联，标注：
- 关联类型（强化/对冲/因果）
- 置信度（高/中/低）
- 如果该关联成立，可能的后续发展


### 阶段 2.8：补充搜索（Delta Retrieve）

对以下情况，进行针对性补充搜索：
- 本周新出现的 emerging 信号：搜索更多背景信息，辅助判断是否为真正的新兴趋势
- 连续 3 天以上出现的主题簇：搜索该主题的最新进展，补充日报可能遗漏的信息
- 关联分析中置信度为"低"的关联：搜索更多证据来提升或否定该关联

补充搜索的结果标注为"周报补充搜索"来源，与日报原始数据区分。
如果补充搜索无显著新发现，简要说明即可，不要为了搜索而搜索。

**论文阅读限制**：补充搜索中如涉及学术论文，浏览摘要不限篇数，但精读全文最多 `budget.paper.weekly_deep_read` 篇，优先选择文献综述（review/survey）。

### 阶段 3：信号状态判定

采用 Topic Emergence Map（TEM）四象限模型（参考 WISDOM 框架），基于两个维度对信号分类：
- **X 轴：当前热度**（topic share 占比）
- **Y 轴：趋势方向**（周环比变化率）

四象限分类：

| 象限 | 热度 | 趋势 | 分类 | 含义 |
|------|------|------|------|------|
| 右上 | 高 | ↑ | **trending（强信号）** | 高关注度且仍在增长 |
| 左上 | 低 | ↑ | **emerging（弱信号）** | 关注度低但增长快，值得重点跟踪 |
| 右下 | 高 | ↓ | **fading（衰退信号）** | 曾经热门但正在降温 |
| 左下 | 低 | ↓ | **declining（潜伏信号）** | 低关注度且仍在下降 |

状态迁移规则：

| 上周状态 | 本周表现 | 新状态 |
|---------|---------|--------|
| trending | 仍高频且增长 | **trending**（持续） |
| trending | 高频但开始下降 | **fading** |
| emerging | 频率上升，进入高热度 | **升级为 trending** |
| emerging | 频率继续低但仍增长 | **emerging**（持续） |
| emerging | 消失 | **降级为 declining** |
| fading | 继续下降 | **fading**（持续）或 **declining**（跌出高热度） |
| fading | 重新增长 | **回升为 trending** |
| declining | 重新出现且增长 | **复苏为 emerging** |
| 无记录 | 本周新出现且 ≥3 天 | **新增 emerging** |
| 无记录 | 本周新出现但仅 1-2 天 | **观察中**（不入 signals） |

对弱信号（emerging），参考 environmental-scanning-foresight 的验证框架评估：
- 来源可信度（高/中/低）
- 独立来源数量
- 放大后的潜在影响

### 阶段 4：注意力流向分析

回答核心问题：**本周关注点在往哪移动？**

- 哪些领域的信息密度在增加？（注意力流入）
- 哪些领域在变安静？（注意力流出）
- 是否有跨领域的关联模式？（如：某材料突破 + 某公司融资 → 指向同一方向）
- 本周最值得关注的 3-5 个发展是什么？（不是最多的，是最重要的）

### 阶段 5：更新 signals

基于阶段 3 的判定结果，生成更新后的 signals 数据。

读取 `${SCAN_CONFIG}`，仅更新其中的 `signals` 区域（trending/emerging/fading/declining 四个列表），将更新后的完整文件写回 `${SCAN_CONFIG}`。

## 约束

- 按主题聚类组织内容，不要按天罗列
- 量化指标必须基于实际数据统计，不要估算
- 如果本周日报不足 3 篇，在概览中说明数据量有限，分析结论置信度较低
- 不要编造日报中不存在的信息
